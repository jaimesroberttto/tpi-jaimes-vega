1. INTRODUCCION

En este trabajo practico integrador se desarrollo una aplicacion en Python orientada a la
gestion de datos de paises. El sistema fue pensado para trabajar con un archivo CSV como
fuente de informacion y permitir al usuario consultar, agregar, actualizar, filtrar, ordenar y
analizar datos de distintos paises mediante un menu en consola.

La propuesta integra contenidos centrales de la materia Programacion 1, especialmente el uso
de listas, diccionarios, funciones, estructuras condicionales y repetitivas, manejo de archivos
y calculo de estadisticas basicas. Ademas, se priorizo una estructura de codigo clara y
modular para facilitar tanto la comprension como el mantenimiento del programa.


2. OBJETIVO DEL PROYECTO

El objetivo principal del proyecto fue desarrollar un sistema en Python que permita gestionar
informacion de paises a partir de un dataset almacenado en formato CSV.

De manera mas especifica, se busco:

- Leer y procesar informacion desde un archivo CSV.
- Representar los datos en memoria usando estructuras adecuadas.
- Permitir operaciones de busqueda, filtrado y ordenamiento.
- Obtener estadisticas relevantes del conjunto de datos.
- Aplicar validaciones para evitar errores durante la ejecucion.
- Utilizar funciones para dividir el problema en tareas mas pequenas y organizadas.


3. MARCO TEORICO

3.1 LISTAS

Las listas en Python son estructuras de datos ordenadas y mutables que permiten almacenar
multiples elementos en una sola variable. En este proyecto se utilizo una lista para guardar
todos los paises cargados desde el archivo CSV. Cada elemento de la lista representa un pais.

El uso de listas fue apropiado porque:

- permiten recorrer facilmente todos los registros,
- facilitan operaciones como agregar elementos,
- permiten aplicar filtros y ordenamientos,
- se integran de forma natural con bucles y funciones.


3.2 DICCIONARIOS

Los diccionarios son estructuras que almacenan pares clave-valor. En este proyecto, cada pais
fue representado mediante un diccionario con las claves:

- nombre
- poblacion
- superficie
- continente

El diccionario fue una estructura adecuada porque permite acceder a cada dato del pais de
forma clara y directa, mejorando la legibilidad del codigo.


3.3 FUNCIONES

Las funciones permiten dividir un programa en bloques reutilizables y organizados, donde cada
bloque cumple una tarea especifica. En este trabajo se utilizaron funciones para:

- cargar y guardar datos,
- validar entradas,
- agregar paises,
- actualizar informacion,
- realizar busquedas,
- aplicar filtros,
- ordenar resultados,
- calcular estadisticas.

La modularizacion mediante funciones mejora la claridad del programa, evita repeticiones de
codigo y hace que el sistema sea mas facil de mantener y corregir.


3.4 CONDICIONALES

Las estructuras condicionales permiten tomar decisiones en funcion de determinadas
condiciones. En el proyecto se utilizaron para:

- verificar si un valor ingresado es valido,
- controlar si un pais ya existe,
- detectar si una busqueda arrojo resultados,
- validar rangos minimos y maximos,
- mostrar mensajes de exito o error.

El uso correcto de condicionales es fundamental para el control del flujo del programa.


3.5 ESTRUCTURAS REPETITIVAS

Las estructuras repetitivas, como `while` y `for`, se utilizaron para:

- mantener el menu principal en ejecucion,
- recorrer listas de paises,
- repetir pedidos al usuario hasta obtener un dato valido,
- mostrar resultados y estadisticas.

Estas estructuras permitieron automatizar tareas y evitar codigo repetitivo.


3.6 ORDENAMIENTOS

Ordenar datos es una operacion muy importante cuando se trabaja con conjuntos de informacion.
En este proyecto se implementaron ordenamientos por:

- nombre,
- poblacion,
- superficie.

Se utilizo la funcion `sorted()` de Python, que permite ordenar listas de forma simple y
eficiente, indicando el criterio mediante funciones anonimas (`lambda`) cuando fue necesario.


3.7 ESTADISTICAS BASICAS

El sistema implementa estadisticas para extraer informacion util del dataset. Entre ellas:

- pais con mayor poblacion,
- pais con menor poblacion,
- promedio de poblacion,
- promedio de superficie,
- cantidad de paises por continente.

Estas operaciones permiten transformar los datos en informacion interpretable para el usuario.


3.8 ARCHIVOS CSV

El formato CSV (Comma Separated Values) permite almacenar informacion tabular en archivos de
texto. En este trabajo se utilizo para guardar el dataset de paises.

Se emplearon las herramientas del modulo `csv` de Python para:

- leer el archivo con `csv.DictReader`,
- escribir el archivo con `csv.DictWriter`,
- conservar una estructura clara basada en encabezados.

El CSV fue una buena eleccion porque es simple, portable y facil de mantener.


4. DESCRIPCION DEL PROBLEMA

La consigna del trabajo plantea desarrollar una aplicacion de consola capaz de gestionar datos
de paises, utilizando como base un archivo CSV. El sistema debe permitir al usuario consultar y
manipular informacion de forma ordenada, clara y segura.

El problema principal consiste en transformar un conjunto de datos crudos en un sistema
interactivo que permita:

- agregar nuevos registros,
- actualizar informacion existente,
- buscar datos especificos,
- aplicar filtros,
- ordenar resultados,
- calcular indicadores generales.


5. ESTRUCTURA DE DATOS UTILIZADA

La estructura principal utilizada fue una lista de diccionarios.

Ejemplo conceptual:

[
    {
        "nombre": "Argentina",
        "poblacion": 45376763,
        "superficie": 2780400,
        "continente": "America"
    },
    {
        "nombre": "Japon",
        "poblacion": 125800000,
        "superficie": 377975,
        "continente": "Asia"
    }
]

Esta combinacion resulta adecuada porque:

- la lista permite almacenar muchos paises,
- el diccionario representa cada pais con sus atributos,
- la estructura es flexible y facil de recorrer,
- permite aplicar filtros, ordenamientos y estadisticas de forma simple.


6. DECISIONES TECNICAS Y ARQUITECTURA

Para organizar el proyecto se separo el codigo en dos archivos principales:

- `main_paises.py`: contiene el menu principal y la interaccion con el usuario.
- `funciones_paises.py`: concentra la logica del sistema, el manejo del CSV, las validaciones,
  los filtros, los ordenamientos y las estadisticas.

Ademas, se incorporo un archivo `paises.csv` como dataset base del proyecto.

Las principales decisiones tecnicas fueron:

- trabajar con una lista de diccionarios en memoria,
- utilizar `csv.DictReader` y `csv.DictWriter` para una lectura y escritura mas clara,
- usar funciones pequenas y especificas,
- incorporar validaciones para evitar errores comunes,
- mantener el sistema en consola para respetar la consigna de Programacion 1.


7. FLUJO GENERAL DEL SISTEMA

El flujo general del sistema puede describirse de la siguiente manera:

1. El programa inicia.
2. Se verifica la existencia del archivo CSV.
3. Se cargan los paises en memoria.
4. Se muestra un menu principal.
5. El usuario selecciona una opcion.
6. El sistema ejecuta la accion correspondiente:
   - mostrar
   - agregar
   - actualizar
   - buscar
   - filtrar
   - ordenar
   - calcular estadisticas
7. Se vuelve al menu principal hasta elegir salir.

En la version final del informe, este apartado puede complementarse con un diagrama de flujo.


8. FUNCIONALIDADES IMPLEMENTADAS

El sistema desarrollado incluye las siguientes funcionalidades:

- Mostrar todos los paises cargados.
- Agregar un nuevo pais.
- Actualizar la poblacion y la superficie de un pais existente.
- Buscar paises por nombre con coincidencia parcial o exacta.
- Filtrar paises por continente.
- Filtrar paises por rango de poblacion.
- Filtrar paises por rango de superficie.
- Ordenar paises por nombre.
- Ordenar paises por poblacion.
- Ordenar paises por superficie.
- Mostrar estadisticas generales del dataset.


9. VALIDACIONES Y MANEJO DE ERRORES

Para mejorar la robustez del sistema, se implementaron distintas validaciones:

- control de campos vacios,
- validacion de numeros enteros positivos,
- control del rango minimo y maximo en filtros,
- mensajes cuando no hay resultados en busquedas o filtros,
- deteccion de lineas invalidas en el CSV,
- mensajes de error y exito claros para el usuario.

Estas validaciones fueron importantes para evitar fallos de ejecucion y para brindar una mejor
experiencia de uso en consola.
